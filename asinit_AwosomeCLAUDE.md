# ASINIT 执行协议

你正在执行 `asinit` 命令。这是一个初始化命令，用于生成项目的执行规范。

## 前置检查

无强制前置条件。specs 文件为可选，根据用户意图自动判断执行模式。

## 唯一动作

更新 `CLAUDE.md` 中的 asinit 协议部分：

1. 若 `CLAUDE.md` 不存在 → 创建新文件，写入下方内容
2. 若 `CLAUDE.md` 存在但无 `<!-- ASINIT START -->` 标记 → 将下方内容插入到文件**最前面**
3. 若 `CLAUDE.md` 存在且有标记 → 只更新 `<!-- ASINIT START -->` 和 `<!-- ASINIT END -->` 之间的内容，**保留文件其他部分不变**

不做任何其他动作。

## 生成内容

写入以下内容（包含标记）：

```markdown
<!-- ASINIT START -->

# 执行协议

---

## 一、标准执行流程

### 模式判断

每次收到开发请求时，判断用户是否表达了使用 specs 开发的意图：

**触发严格执行模式的情况：**
- 用户提及了 specs 相关文件（如 `requirements.md`、`design.md`、`tasks.md`）
- 用户引用/索引了任何 specs 文档
- 用户明确要求按 specs 执行
- 用户提到"按规范"、"按设计文档"、"按任务列表"等类似表述

**判断结果：**
- 用户表达了使用 specs 开发 → 进入 [严格执行模式](#严格执行模式)
- 用户未提及 specs → 进入 [通用开发模式](#通用开发模式)

### 严格执行模式

当存在 specs 文档时，必须按序执行以下 5 步。

**步骤 1：加载 Specs（强制）**

定位 specs 目录，**强制加载所有文件**：
- `requirements.md` ← **本次任务的真理，必须严格执行**
- `design.md`
- `tasks.md`
- 项目级/任务级 PRD（若存在）

**重要：** Specs 是唯一权威来源，所有实现必须严格遵循 specs 定义。

**步骤 2：单 Task 执行**
- 只选择 tasks 中**尚未完成**的 task
- 一次只实现**一个** task（禁止合并或提前实现）
- 若 task 未要求测试，必须编写增量测试代码

**步骤 3：强制测试**
- 实现后必须运行测试
- 测试失败必须修复后重跑
- 测试不通过禁止进入下一阶段

**步骤 4：Gemini 验收（使用 subagent）**

测试通过后调用 Gemini subagent 进行批判性验收：
- 角色：严格、挑刺、站在该功能最佳实践角度的代码审核员
- 验收标准：specs + PRD
- 必须给出具体的改进建议

根据 Gemini 返回结果判断是否需要修改：
- 若需修改：执行修改 → 重跑测试 → 二次 Gemini 验收（仅返回 PASS/FAIL，节省 token）
- 若不通过：继续修改直到通过

**步骤 5：提交与结束**

仅在验收 PASS 后：
1. 编写 commit message（**描述使用中文**，本地 commit，禁止 push）
2. 更新 `tasks.md` 中对应 task 的完成状态
3. 停止执行，等待下一次指令

### 通用开发模式

当不存在 specs 文档时，按以下流程执行。

**步骤 1：理解需求**
- 分析用户请求，明确开发目标
- 若需求不清晰，主动询问澄清
- 评估影响范围和实现方案

**步骤 2：实现开发**
- 遵循项目现有代码风格和架构
- 编写清晰、可维护的代码
- 适当添加注释说明关键逻辑

**步骤 3：测试验证**
- 编写或更新相关测试
- 运行测试确保功能正常
- 测试失败必须修复

**步骤 4：提交**
1. 编写 commit message（**描述使用中文**，本地 commit，禁止 push）
2. 简要说明完成的工作

---

## 二、规范约束

<!-- CONSTRAINTS START -->

### 测试规范

1. **语言要求**：测试文件中禁止使用中文注释和中文字符，所有注释、变量名、测试描述必须使用英文
2. **测试文件组织**：
   - 使用项目根目录下的统一测试目录（如 `tests/` 或 `__tests__/`）
   - 禁止在各功能模块文件夹内创建独立的测试文件
   - 若项目尚无全局测试目录，需在根目录新建 `tests/` 文件夹统一管理
   - 测试文件命名应与被测模块对应，如 `tests/user.test.ts` 对应 `src/user.ts`

<!-- CONSTRAINTS END -->

<!-- ASINIT END -->
```

## 完成输出
